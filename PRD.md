## 1. 背景與機會

- 台北捷運**不公開列車即時位置**，官方 App 只有各站到站倒數；一般乘客與鐵道愛好者沒有任何「看得到全網列車在哪」的視覺化工具。
- Railisland（軌島）已驗證「時刻表推演＋官方到站看板校正」的做法在台灣可行且受歡迎，但它是全台鐵道大雜燴，北捷只是配角（示意車、輕量資訊卡）。
- 交通部 TDX 平台提供北捷完整開放資料（路網幾何、逐站時刻表、班距、即時到站看板），資料取得零成本。
- **機會**：做一個「只做北捷、但把北捷做到最好」的即時模擬地圖——更精緻的路網、逐班時刻、更準的校正、更完整的車站互動。

## 2. 產品願景

> **打開網頁，就看見整個台北捷運正在呼吸。**
> 每一班列車依真實時刻表在地圖上奔跑，點一站看板、點一車跟隨，到站時間與月台看板同步。

產品名稱：**捷奏**（取「捷運的節奏」之意）。定位：免費、無登入、開瀏覽器即用的 Web 應用（PWA 可安裝為後續目標）。

## 3. 目標用戶與情境

| 用戶 | 情境 | 需求 |
|---|---|---|
| 通勤族 | 出門前／月台等車時打開 | 「我這站下一班還有多久？現在車到哪了？」 |
| 鐵道／地圖愛好者 | 掛著看、探索路網 | 看全網列車交錯、跟一班車跑完全程、倍速快轉一整天 |
| 教學／展示 | 課堂、簡報、大螢幕展示 | 直覺的城市脈動視覺化，零操作也好看 |

**明確不是**：導航工具（不做轉乘規劃）、官方資訊管道（誤點以官方為準）。

## 4. 核心價值與差異化

1. **全網同框**：北捷 5 主線＋2 支線所有列車同時運行，非單站查詢。
2. **逐班真實時刻**：大多數路線用當日逐站時刻表推演（非單純班距動畫）；文湖線以官方班距合成並明確標示。
3. **即時校正**：營運時段以官方到站看板持續校正，目標到站誤差中位數 ≤ 45 秒。
4. **可玩性**：跟車視角、時間旅行（倍速／回放）、車站看板——把「等車」變成「看車」。

## 5. 功能需求

優先級定義：P0 = MVP 上線必要；P1 = 上線後第一波；P2 = 後續探索。

### P0 — MVP

| # | 功能 | 說明 | 驗收要點 |
|---|---|---|---|
| F1 | 路網地圖 | 北捷 5 線（文湖、淡水信義、松山新店、中和新蘆、板南）＋新北投、小碧潭支線，官方線色，車站節點與站名（依縮放分級顯示） | 幾何無斷裂、無鋸齒繞行；縮放流暢 |
| F2 | 列車模擬 | 依當日時刻表（平日／週六／週日）推演所有列車位置，含出站加速／進站減速 | 1× = 真實速度；開站即現在時刻；與時刻表誤差 < 5s（未校正前、模擬層面） |
| F3 | 時間控制 | 播放／暫停、倍速 1×/10×/30×/60×、時間軸拖曳、「回到現在」 | 空白鍵暫停；拖曳跨午夜正常 |
| F4 | 車站看板 | 點車站 → 面板顯示雙向未來班次（終點站、預計到站倒數） | 營運時段顯示校正後時間；收班顯示「今日收班」 |
| F5 | 跟車模式 | 點列車 → 鏡頭跟隨＋資訊卡（線別、方向、下一站、終點、當前時速） | 跟隨中可拖曳地圖、一鍵回到列車；到終點自動解除 |
| F6 | 即時校正 | 營運時段輪詢官方到站看板，校正模擬相位 | 到站誤差中位數 ≤ 45s（抽站人工對板驗收）；文湖線標示「班距推算」 |
| F7 | 手機可用 | RWD、觸控操作、省電降幀 | iPhone SE 尺寸可完整操作；中階 Android ≥ 30fps |

### P1 — 第一波迭代

| # | 功能 | 說明 |
|---|---|---|
| F8 | 深色模式 | 跟隨系統＋手動切換；夜間底圖 |
| F9 | 分享深連結 | 目前視角／時間／跟隨中列車打包成 URL，開啟即還原 |
| F10 | 營運告警 | 北捷官方公告／異常橫幅（TDX Alert） |

### P2 — 探索方向（不承諾）

環狀線／機捷／淡海輕軌擴充（跨營運商）、收集玩法（完乘章）、放空／劇場模式、車站出口與周邊資訊、站內時刻表匯出。

## 6. 非功能需求

| 類別 | 要求 |
|---|---|
| 效能 | 首屏 LCP ≤ 2.5s（4G／中階手機）；桌機 60fps、手機 ≥ 30fps；初始下載（不含底圖磚）≤ 1.5MB gzip |
| 可用性 | 部署於用戶既有 Vercel 帳號；TDX 掛掉時退化為純時刻表模擬並顯示狀態 |
| 相容 | Chrome / Edge / Safari / Firefox 近兩版；iOS Safari 為一級目標 |
| 成本 | 月固定成本目標 ≈ NT$0（Vercel Hobby 免費層＋TDX 免費方案）＋網域費；流量若超出免費層再評估升級 |
| 合規 | 標示資料來源（交通部 TDX／臺北捷運公司）與底圖版權；本專案程式碼以 MIT 釋出（記取 Railisland 無授權的教訓） |
| 免責 | 明示「時刻表推演＋看板校正，非官方即時位置，請以現場資訊為準」 |

## 7. 成功指標（P0 上線後 4 週）

- **準確**：抽 10 站 × 尖離峰各一次人工對板，到站誤差中位數 ≤ 45s、P90 ≤ 90s。
- **效能**：Lighthouse Performance ≥ 85（行動）；實機 iPhone 12 / 中階 Android 不掉幀卡死。
- **參與**：平均停留 ≥ 3 分鐘；跟車功能使用率 ≥ 30%（簡易匿名事件統計）。
- **口碑**：公開發佈文（Threads／PTT）自然回饋正向；GitHub 開源獲得社群 star。

## 8. 範圍外（明確不做）

- 轉乘／票價／路徑規劃（官方與 Google Maps 已做好）
- 帳號系統、任何個資蒐集
- 台鐵／高鐵（Railisland 已覆蓋；我們押注「單一系統做深」）
- 原生 App（PWA 為止）

## 9. 風險與緩解

| 風險 | 等級 | 緩解 |
|---|---|---|
| 北捷逐站時刻表無車次號，串班演算法可能誤串 | ~~中~~ 低（Spike 證實可按交路串班） | M1 設驗證器（串接率、單調性檢查）；容差可調；誤串退化為班距合成 |
| TDX 到站看板欄位／單位與文件不符 | ✅ 已解（Spike 實測：事件式看板） | 校正演算法改為進站事件比對（SDD §6） |
| LiveBoard 為事件式、單次樣本稀疏（約 30 筆/次） | 低 | 每 20s 輪詢連續累積事件＋中位數平滑；模擬本身即為底線 |
| 文湖線無逐班時刻，精度體感差 | 中 | UI 明確標示「班距推算」；看板校正拉回相位 |
| TDX 限流／中斷 | 低 | Worker 集中快取（使用者數與 API 呼叫數脫鉤）；斷線退化純模擬 |
| 路網幾何品質（支線、機廠段雜線） | 中 | 管線自動清理＋debug 疊圖人工檢查 |
| 單人開發量（估 ~32 人日） | — | 垂直切片：先淡水信義線端到端跑通，再鋪全網 |
